Software Design and Engineering

Achieved Outcomes

  • Gathering Requirements (#1)
    Successfully modified the customer frontend based on a single client's needs and began establishing admin frontend requirements. Documented prototype versions and reasons for design decisions to align with stakeholder input.
  • Professional Documentation (#2)
    Produced detailed website wireframes and updated portions of the Software Design Document (SDD). These documents outline new API calls, functional enhancements, and future directions for the MEAN application.
  • Migrating a Wix Site to MEAN (#3)
    Converted core functionality from a Wix-based solution to a MEAN stack application, laying the groundwork for a component-based editor that will evolve further in subsequent enhancements.

Future Improvements

  • Enhanced Security (#5)
    While the application maintains basic security, there is a need for a more comprehensive approach to API call protection and advanced authorization checks. Future updates will include robust security measures to fully align with outcome #5.
  • SDD Overhaul
    The Software Design Document requires additional updates as new API calls and backend changes are introduced in the next phase of development.
  • Refining Admin MVP
    Although a minimum viable product exists for FEC editing, further improvements—such as expanded editing capabilities and deeper security integrations—are planned to optimize the admin experience.

Algorithms and Data Structures

Achieved Outcomes

  • Algorithmic Solutions (3)
    Refactored the application’s frontend model and controller patterns, as well as the API model hierarchy, to streamline CRUD operations and improve data delivery mechanisms. This overhaul significantly simplified algorithms for handling page objects and Frontend Express Components (FECs).
  • Innovative Computing (4)
    Created and leveraged a “base model” and “page model” to enable dynamic page rendering. This approach not only supports modular and DRY code practices, but also allows the Express site to generate pages based on a single document, minimizing the need for manual controller logic.

Future Improvements

  • Security Mindset (5)
    Although the groundwork for role-based authorization was laid, full integration of security measures remains incomplete. Plans include enhancing API protections and incorporating robust route security, ensuring only minimal, relevant data is shared with authorized roles.
  • Deeper Admin Frontend Integration
    While the admin solution evolved during this enhancement, additional improvements—such as more seamless form updating and an earlier discovery of optimal data flow—will be pursued in the next iteration.

Databases

Achieved Outcomes

  • Algorithmic Solutions (3)
    Consolidated the application's routes under .route('/components/:componentType/:id'), streamlining data flow and reducing redundant logic, which allowed the Angular and Express sites to load page elements dynamically.
  • Innovative Computing (4)
    Employed new techniques in user roles and RBAC to deliver scalable, secure solutions. By centralizing component logic in a single route, the application further supports future feature expansions.
  • Security Mindset (5)
    Implemented robust Role-Based Access Control (RBAC) and JWT-based user roles, ensuring that sensitive data remains secure. The system now filters data at the API level based on user privileges, preventing unauthorized access.

Future Improvements

  • Additional Refinements
    While the core security and data structure goals were met, further refactoring is needed to maintain clarity, including database models for storing roles and granular page access.
  • Incomplete Image Select Tool
    The final milestone for full editability remains unfinished. Completing this feature will allow all Front-End Components (FECs) to be edited via the Angular frontend, maximizing the programmatic approach to page generation.
  • Maintainability & Optimization
    Plans include additional testing for performance under heavier loads and ongoing improvements to keep code maintainable as new features are introduced.
